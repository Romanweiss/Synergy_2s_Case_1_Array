Подробное описание процесса решения задачи
1. Анализ исходного решения
- Изначально программа работала с режимами `int` и `float`, а выбор режима происходил через аргумент командной строки.
- Логика вычисления суммы отрицательных элементов между минимумом и максимумом уже была реализована корректно.

2. Постановка доработки
- Требовалось изменить сценарий запуска: сначала пользователь выбирает тип чисел, и только затем выполняются генерация массива и вычисления.
- Дополнительно нужно сохранить поддержку трёх режимов: `int`, `float`, `mixed`.

3. Архитектурное решение
- Добавлена отдельная функция пользовательского выбора типа (`choose_number_type`).
- Выбор типа вынесен в начало выполнения `main()`.
- Генерация массива и расчёты запускаются только после валидного ввода.

4. Реализация режима смешанных типов
- Учтено ограничение модуля `array`: один массив `array` хранит только один тип (`int` или `float`).
- Для режима `mixed` применён стандартный список `list`, который допускает одновременно `int` и `float`.

5. Сохранение и развитие вычислительной логики
- Алгоритм расчёта не изменён по смыслу:
  - находится минимум и максимум;
  - определяется диапазон между их индексами;
  - суммируются отрицательные элементы в этом диапазоне.
- Добавлено форматирование вывода, чтобы вещественные числа отображались с двумя знаками после запятой.

6. Обновление документации
- `README.md` обновлён под новый интерактивный сценарий работы.
- Добавлены описание режимов, ограничений по `array`, шагов работы и пример запуска.

Структура программы и ответственность функций
1. `choose_number_type()`
- Показывает пользователю меню выбора режима (`int`, `float`, `mixed`).
- Проверяет корректность ввода.
- Возвращает выбранный тип только после валидного значения.

2. `random_int()`
- Генерирует одно случайное целое число в диапазоне `[-1000, 1000]`.

3. `random_float()`
- Генерирует одно случайное вещественное число в диапазоне `[-1000, 1000]`.
- Округляет результат до 2 знаков после запятой.

4. `random_mixed_number()`
- Случайно выбирает, какое число вернуть в текущей позиции: `int` или `float`.
- Используется при построении смешанного массива.

5. `generate_array(size, number_type)`
- Генерирует массив нужного формата на основе выбранного типа:
  - `int` -> `array('i')`;
  - `float` -> `array('d')`;
  - `mixed` -> `list`.

6. `as_list(values)`
- Приводит структуру данных к списку Python для унифицированной дальнейшей обработки.
- Если на входе `array`, преобразует через `.tolist()`.

7. `calculate_negative_sum_between_extremes(values)`
- Вычислительное ядро программы.
- Находит минимальный и максимальный элементы, их индексы.
- Определяет промежуток между ними.
- Считает сумму отрицательных элементов внутри этого промежутка.
- Возвращает минимум, максимум и итоговую сумму.

8. `format_number(value)`
- Форматирует число для печати:
  - `float` -> с 2 знаками после запятой;
  - `int` -> без изменений.

9. `format_array(values)`
- Формирует строковое представление массива для аккуратного вывода в консоль.
- Использует `format_number` для единообразного отображения элементов.

10. `main()`
- Точка входа программы.
- Последовательность действий:
  1) получение типа через `choose_number_type`;
  2) генерация размера массива;
  3) генерация массива;
  4) запуск расчётов;
  5) вывод результатов.

Пошаговый сценарий работы программы (для отчёта)
1. Пользователь запускает `python main.py`.
2. Программа запрашивает тип чисел.
3. До корректного ввода дальнейшие действия не выполняются.
4. После выбора типа генерируется массив случайной длины от 10 до 30.
5. Рассчитываются минимум и максимум.
6. Вычисляется сумма отрицательных элементов между ними.
7. Результат выводится в консоль вместе с исходным массивом.

Итог
Программа реализует интерактивный и понятный пользователю сценарий работы, поддерживает целые, вещественные и смешанные массивы, а также сохраняет единый корректный алгоритм вычисления суммы отрицательных элементов между экстремумами.
